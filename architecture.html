Where The Magic Lives

The complete data pipeline of the YTMusic+ Engine

Phase 1: Ingestion

The Data Source (Last.fm → Supabase)

The system doesn't fetch "new" music randomly from the internet. It relies on your personal history. As you listen to music on Spotify/YouTube, Last.fm logs it (scrobbling).

Our Edge Function (`sync-lastfm`) wakes up, pulls your latest plays from the Last.fm API, and saves them permanently into your Supabase Database (filling the tracks and plays tables).

Phase 2: The Brain

Where The Algorithm Lives (Supabase RPC)

When you click "Generate" in React, it doesn't do the math on the frontend. React sends a request to a PostgreSQL Remote Procedure Call (RPC) named generate_smart_shuffle.

This is where the algorithm lives. Doing this inside the database is lightning fast because the data never has to travel across the internet to be calculated.

The database calculates the Affinity Score for thousands of your tracks instantly, applies your Vibe Filter (e.g., Techno), sorts them into the 45% / 35% / 20% buckets, randomizes them, and shoots exactly 50 tracks back to React.

Supabase RPC
PL/pgSQL
Phase 3: The Translation

Finding the YouTube Videos (`resolve-yt-ids`)

We now have 50 great tracks, but they are just text (e.g., "Daft Punk - One More Time"). YouTube doesn't know what to do with that. We need the specific 11-character YouTube Video IDs.

React sends the 50 tracks to the `resolve-yt-ids` Edge Function. This function uses a stealthy scraper (youtube-sr) to search YouTube for free, completely bypassing Google's strict quota limits.

Aggressive Caching: Once it finds the video ID, it saves it back into your Supabase database. The next time this song rolls in a shuffle, it skips the search entirely and loads instantly.

Phase 4: The Execution

Pushing to YouTube Music

Now React holds 50 exact YouTube Video IDs. The final step is making them playable.

React uses your Google OAuth Access Token to talk directly to the YouTube Data API v3. It executes two fast commands:

1. POST /playlists (Creates "YTMusic+ Discovery")
2. POST /playlistItems (Loops 50 times, adding each video ID)

Because YouTube and YouTube Music share the same backend, the moment this finishes, the playlist instantly appears in your YouTube Music app, ready to play.
